Managing Tasks



The various operations, allowing you to have a certain amount of control over the execution of tasks, are listed below. Tasks are assigned different stages as they proceed through processing. Those stages include "Setup", "Queued", "Cluster" and "Post Processing" and "Completed" stages. The Setup stage sets up the files to be processed. When a task is Queued it is waiting to be processed. The Cluster (also referred to as the running stage) stage is the scientific processing stage. The Post Processing stage saves the results back to the CBRAIN data provider. The Completed stage indicates that the processing has been completed. Should a task fail, failure recovery code can be triggered.
  • Operations for Queued Tasks
    • Hold queued tasks
    • Release held tasks
  • Operations for Running Tasks
    • Suspend running task
    • Resume suspended tasks
  • Operations for failed tasks
    • Trigger failure recovery code
  • Operations for completed tasks
    • Restart tasks at "Setup" stage
    • Restart tasks at "Cluster" stage
    • Restart tasks at "Post Processing" stage
  • Operations for All Tasks
    • Duplicate selected tasks and start them anew
    • Terminate selected tasks
    • Remove selected tasks
