Introduction To Access Profiles

Access Profiles are way to simplify user access management to sets of projects.

An Access Profile is basically a set of arbitrary projects. When users are created
or modified, they can be assigned to one (or several) Access Profiles, and thus
they become a member of all their projects immediately.

Access Profiles and all their properties are never visible to normal users.

Creating an Access Profile

When creating a new Access Profile, the admin can set a color to help
differentiate it from other Access Profiles. Projects can also immediately
be assigned to it.

Deleting an Access Profile

Removing an Access Profile from a user will remove all the projects at once
(except those that may be part of other Access Profiles).

Adjusting an Access Profile: adjusting the projects

Adjusting the set of project in an Access Profile will (by default) immediately
affect all users associated with that Access Profile; however the
admin can select which users will be affected by the change using the checkboxes
at the bottom of the project list.

Adjusting an Access Profile: adjusting the users

Adjusting the list of users associated with an Access Profile will affect
all those users: they will either become members of all the projects in
the profile (if the user is added) or might lose access to the project
(if removed, and the project is in no other access profile that the
user has access to).
